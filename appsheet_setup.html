คู่มือเปิดระบบล็อกอินจริง (AppSheet)

ทำไมต้องตั้งค่านี้

พอร์ทัล (เว็บนิ่ง) ใช้ "รหัสอำนวยความสะดวก" เพื่อแยกหน้าตามบทบาทเท่านั้น — ข้อมูลส่วนตัวจริง (คะแนนรายคน/ทั้งโรงเรียน) อยู่ในระบบกลาง AppSheet ซึ่งจะแสดง "เฉพาะข้อมูลในสิทธิ์ของผู้ใช้" ได้ก็ต่อเมื่อ เปิด Require Sign-In และผูกผู้ใช้กับบทบาทเรียบร้อย

เป้าหมาย: ผู้ปกครองเห็นเฉพาะลูกตัวเอง · โรงเรียนเห็นเฉพาะโรงเรียนตัวเอง · เจ้าหน้าที่/ผู้บริหารเห็นตามสิทธิ์ — ทั้งหมดทำงานด้วย Security Filter ตามอีเมล (USEREMAIL()) ที่ตั้งไว้แล้ว
ขั้นตอนตั้งค่า (ทำครั้งเดียว)
1
เปิด Require Sign-In
ในตัวแก้ไขแอป (AppSheet editor) ไปที่ Settings ⚙ → Security → Require Sign-In แล้วสลับเป็น ON · เลือกผู้ให้บริการล็อกอิน Google (และ/หรืออีเมลอื่นตามต้องการ)
2
เพิ่มผู้ใช้ที่อนุญาต
ไปที่ Users → Manage Users (Add users) ใส่อีเมลผู้ใช้จริง: เจ้าหน้าที่ · ผู้บริหาร · ผู้ดูแลโรงเรียน · ผู้ปกครอง — ระบบจะส่งคำเชิญให้ล็อกอิน
3
ผูกอีเมลกับบทบาทในตาราง Roles
เปิดตาราง Roles เพิ่มแถว: อีเมลผู้ใช้ + บทบาท ให้ตรงกับที่ Security Filter ใช้ตรวจ เช่น ผู้บริหาร (Executive) / เจ้าหน้าที่ (Staff) / โรงเรียน / ผู้ปกครอง · สำหรับผู้ปกครอง ให้อีเมลตรงกับช่อง "อีเมลผู้ปกครอง" ในตาราง Students
4
ตรวจ Security Filter ของตารางข้อมูล
ที่ Data → (ตาราง Scores/Students) → Security Filter ตรวจให้ใช้สูตรอิงอีเมลผู้ใช้ (ตัวอย่างด้านล่าง) เพื่อให้แต่ละคนเห็นเฉพาะแถวของตน
OR( [Students].[อีเมลผู้ปกครอง] = USEREMAIL(), IN(USEREMAIL(), SELECT(Roles[อีเมลผู้ใช้], OR([บทบาท]="ผู้บริหาร (Executive)", [บทบาท]="เจ้าหน้าที่/ครู (Staff)"))) )
5
บันทึก แล้วทดสอบ
กด Save → Sync · ทดสอบด้วยปุ่ม Preview as (ดูในมุมผู้ใช้) โดยใส่อีเมลผู้ปกครอง 1 ราย แล้วยืนยันว่าเห็นเฉพาะข้อมูลของลูกตน · จากนั้นลองล็อกอินจริงด้วยบัญชีผู้ใช้ 1 บัญชี
ใครเห็นอะไร (สรุปสิทธิ์)
บทบาทเห็นข้อมูลทำอะไรได้
ผู้ปกครอง / นักเรียนเฉพาะบุตรหลานของตนดูคะแนน · เกียรติบัตร
โรงเรียนเฉพาะนักเรียนในโรงเรียนตนดูภาพรวม · ดาวน์โหลดรายงาน
เจ้าหน้าที่ / ครูทุกคนที่รับผิดชอบลงทะเบียน · กรอกผล · ออกลิงก์
ผู้ดูแลระบบ / ผู้บริหารทั้งระบบจัดการผู้ใช้/บทบาท · ดูเชิงลึก
ข้อควรระวัง
อย่าตั้งแอปเป็น Public (ไม่ต้องล็อกอิน) เพราะจะทำให้ Security Filter ตามอีเมลไม่ทำงาน และข้อมูลเด็กอาจหลุดถึงคนนอก · ให้ Require Sign-In = ON เสมอ
การเปิด Require Sign-In และการเพิ่ม/แชร์ผู้ใช้ เป็นการตั้งค่าความปลอดภัย ผู้ดูแลระบบควรเป็นผู้กดยืนยันเอง (ระบบอัตโนมัติจะไม่กดสลับให้)
